# Dental Practice Website - "Let 'em Shine"

A premium, responsive dental website featuring a "Clean and Glowing" Sky Blue and Teal design system. Built with React, Tailwind CSS, and Supabase.

## 🚀 Quick Start

1.  **Install Dependencies:**
    ```bash
    npm install
    ```
2.  **Run Locally:**
    ```bash
    npm run dev
    ```

## 🛠 Tech Stack

*   **Frontend:** React, Vite, Tailwind CSS v4
*   **Animation:** Motion (formerly Framer Motion)
*   **Icons:** Lucide React
*   **Backend/Database:** Supabase (PostgreSQL)
*   **Components:** Shadcn/UI

## ☁️ Backend Setup (Supabase)

This project requires a Supabase backend for the scheduling system.

1.  **Create a Project:** Go to [supabase.com](https://supabase.com) and create a new project.
2.  **Get Credentials:** Go to Project Settings -> API. Copy:
    *   `Project URL`
    *   `anon` public key
3.  **Environment Variables:**
    Create a file named `.env.local` in the root directory:
    ```env
    VITE_SUPABASE_URL=your_project_url_here
    VITE_SUPABASE_ANON_KEY=your_anon_key_here
    ```
4.  **Create the Database Table:**
    Go to the **SQL Editor** in your Supabase dashboard and run this query to create the bookings table:

    ```sql
    create table public.bookings (
      id bigint generated by default as identity primary key,
      created_at timestamp with time zone default timezone('utc'::text, now()) not null,
      date text not null,
      time text not null,
      service text not null,
      name text, -- Optional: add if you expand the form
      email text, -- Optional: add if you expand the form
      status text default 'pending'
    );

    -- Enable Row Level Security (RLS) is recommended for production
    alter table public.bookings enable row level security;

    -- Policy: Allow anyone to create a booking
    create policy "Enable insert for all users"
    on public.bookings for insert
    with check (true);
    
    -- Policy: Only allow admins to view (You'll need to set up Auth for this later)
    -- For now, purely for development/testing, you might allow select:
    -- create policy "Enable read for all users" on public.bookings for select using (true);
    ```

## 📦 Deployment

Recommended hosting: **Vercel** or **Netlify**.

1.  Push your code to a GitHub repository.
2.  Import the repository into Vercel/Netlify.
3.  **Crucial:** Add the `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` to the "Environment Variables" section in the hosting dashboard.
